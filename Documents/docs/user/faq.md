# 常见问题

面向用户的常见问题解答。技术细节请查阅 [详细说明 FAQ](/guide/faq)。

## 同步相关

### 同步后看不到任何账单

1. 确认同步完成且无错误提示
2. 切换到 **账单** 页检查筛选条件
3. 进入 **设置 → 数据** 点击 **重建索引**
4. 仍为空 → 尝试 **全量同步**

### 一直要求输入验证码

- 切到 **设置 → 验证码 → 远程 OCR**
- 确认 `python3 Scripts/run_ocr_server.py run 5000` 已启动
- 浏览器访问 `http://127.0.0.1:5000/swagger` 看到接口即正常

### 同步到一半卡住

- 检查 `logs/` 下当天的日志，搜索 `ERROR`
- 大概率是 `Captcha failed`，可切手动模式重试
- 网络问题：校园网 vs 外网切换会中断

## 统计相关

### 今天的消费没显示

- 服务器返回的"今天"是校园服务器时区
- 跨午夜的交易按服务器时间归属
- 凌晨 0:00-6:00 的消费可能归到前一天

### 分类不对

应用的分类规则来自 `Classifier/PositionTranslator.cs`（位置关键词 → 分类）。如果发现某商户分类错误，可以：

1. 在 **统计 → 分类** 中点击商户名
2. 选择 **纠正分类**
3. 下次同步时生效

### 月度对比图只显示 1 个月

需要至少 1 个月的同步历史才会绘制。首次安装后请等待。

## 数据相关

### 忘记启动密码

很遗憾，启动密码用于派生 `identities.enc` 的加密密钥，**无法找回**。但账单数据 `bills.db` 是明文的，可手动用 DB Browser for SQLite 打开查看。

解决：删除 `identities.enc`，重新添加账号。

### 应用启动闪退

1. 删除 `cache/` 目录
2. 检查 `logs/latest.log` 的最后 50 行
3. 常见原因：Sqlite 文件损坏 → 从快照恢复

### 切换电脑后账单没了

`Data/` 目录不会自动同步到云端。在新电脑上：

1. 安装应用
2. 从备份（快照或手动复制）恢复 `Data/`
3. 设置启动密码为**原来的密码**（重要，否则账号解不开）

## UI 相关

### 字体太小/太大

**设置 → 外观 → 字体大小**。可选小/中/大/特大。

### 主题不会跟随系统

**设置 → 外观 → 主题 → 跟随系统**。
