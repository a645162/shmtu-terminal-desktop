# 怎么同步账单

同步是把校园一卡通服务器上的消费记录拉取到本地的过程。本节讲三种同步模式、验证码选择和常见问题。

## 三种同步模式

应用在 **同步** 弹窗中提供三种模式：

| 模式 | 行为 | 适用场景 |
|---|---|---|
| **增量同步** | 从上次同步点继续往后拉取，遇到 5 条连续已存在条目即早停 | 日常使用，速度快 |
| **全量同步** | 从第 1 页开始翻页直到翻完 | 数据缺失时补齐历史 |
| **单账号同步** | 只同步当前选中的账号，跳过其他账号 | 多账号情况下只想更新某一个 |

## 同步流程图

```
选择账号 → 选择模式 → 加载验证码
   ↓
解析验证码（手动 / 远程 OCR）
   ↓
登录 CAS 获取 Cookie
   ↓
按页请求账单 HTML
   ↓
HTML 解析为 BillItemInfo
   ↓
去重（按交易号）→ 写入数据库
   ↓
推送进度 → 更新 UI
```

## 选择验证码模式

**设置 → 验证码** 中有 3 种模式：

- **手动输入**：每次弹出验证码图片，用户在 30 秒内输入字符。稳定但繁琐。
- **远程 OCR**：调用本地或局域网内的 ONNX OCR 服务（默认 `http://127.0.0.1:5000`）。识别率约 90%，但完全免输入。
- **本地 ONNX**：直接在桌面进程内加载 ONNX 模型推理，零网络开销。首次使用需要下载 ~170MB 模型。

> 验证码识别失败是同步失败的最常见原因。遇到 `Captcha failed` 错误时，先切回手动模式确认是网络问题还是识别问题。

## 同步失败怎么办

| 现象 | 原因 | 解决 |
|---|---|---|
| `Captcha failed` | 验证码识别错误 | 切回手动模式重试几次 |
| `CAS 登录失败` | 密码错误 / 校园账号被锁定 | 确认账号信息 |
| `网络超时` | 校园网/外网切换 | 检查网络，重试 |
| `JSON 反序列化错误` | 服务端返回了维护页 | 等几分钟再试 |
| 同步卡在 0% | 端口 5000 未启动 | 启动 `run_ocr_server.py` |

## 同步频率建议

- **日常**：每天 1-2 次增量同步即可
- **补齐历史**：每月 1 次全量同步
- **更换设备后**：先全量同步一次
