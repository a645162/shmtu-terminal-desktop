# 设置与数据

本章说明每类设置的含义、推荐值以及对数据的影响。

## 设置分类

应用设置按功能分组：

| 分组 | 内容 |
|---|---|
| **外观** | 主题、字体、动画 |
| **同步** | 默认模式、并发、早停 |
| **验证码** | 手动 / 远程 / 本地 ONNX |
| **数据** | 备份路径、快照、导入/导出 |
| **安全** | 修改启动密码、加密状态 |
| **日志** | 日志级别、查看日志 |
| **关于** | 版本、更新日志、开源许可 |

## 外观

| 项 | 选项 | 推荐 |
|---|---|---|
| **主题** | Light / Dark / 跟随系统 | 跟随系统 |
| **字体大小** | 小 / 中 / 大 / 特大 | 中（默认） |
| **动画** | 开 / 关 | 开（性能允许时） |
| **语言** | zh-CN / en-US（待支持） | zh-CN |

> 主题与字体变更实时生效，不需重启。

## 同步

| 项 | 类型 | 默认 | 说明 |
|---|---|---|---|
| **默认模式** | enum | incremental | 同步弹窗的默认勾选 |
| **并发账号数** | int 1-5 | 1 | 同时同步的账号数（建议 ≤ 3） |
| **早停阈值** | int 1-20 | 5 | 连续多少条已存在即停 |
| **最大翻页数** | int 1-1000 | 100 | 防止无限翻页 |
| **超时（秒）** | int 10-120 | 30 | HTTP 请求超时 |
| **失败重试** | int 0-5 | 3 | 网络失败重试次数 |

### 同步模式详解

- `incremental`：从 `sync_state.last_number` 开始
- `full`：从第 1 页开始
- `single_account`：只同步当前账号

## 验证码

| 模式 | 优点 | 缺点 | 适用 |
|---|---|---|---|
| **手动** | 100% 准确 | 每次需输入 | 网络不稳/不想配 OCR |
| **远程 OCR** | 完全免输入 | 需启动服务，~90% 准确 | 推荐：日常使用 |
| **本地 ONNX** | 零网络开销 | 首次需下载 170MB 模型 | 离线/隐私场景 |

### 远程 OCR 配置

- **服务地址**：默认 `http://127.0.0.1:5000`
- **超时**：30 秒
- **重试**：3 次
- **备用地址**：可填多个，逗号分隔，自动 failover

### 本地 ONNX 配置

- **模型目录**：`ocr/models/`，首次运行自动下载
- **GPU 加速**：勾选启用 DirectML EP（Windows）或 CUDA EP（Linux）
- **推理线程数**：默认 2

## 数据

### 备份路径

应用数据默认位于：

| 平台 | 路径 |
|---|---|
| Windows | `%APPDATA%\shmtu-terminal-desktop\Data` |
| Linux | `~/.local/share/shmtu-terminal-desktop/Data` |
| macOS | `~/Library/Application Support/shmtu-terminal-desktop/Data` |

可在此改为自定义路径（如 Dropbox 同步目录实现自动云备份）。

### 快照

- **手动创建**：点击 **创建快照** → 命名 → 确认
- **自动创建**：可设每周/每月定时（开发中）
- **保留数量**：默认 10 个，超出自动删除最旧
- **恢复**：选择快照 → **恢复** → 应用重启

### 导入

支持导入：

- **CSV**：必须含 `number,amount,position,server_time` 字段
- **JSON**：`[{number, amount, ...}, ...]` 数组
- **钱迹 CSV**：自动识别"钱迹"记账 App 格式

> 导入会自动去重（按 `number`），不会产生重复。

### 导出

| 格式 | 字段 | 用途 |
|---|---|---|
| **CSV** | 全字段 | Excel 打开分析 |
| **JSON** | 全字段 + 嵌套 | 二次处理 |
| **钱迹 CSV** | 适配字段 | 导入"钱迹"App |

### 清空与重建

**危险操作**，会备份 `bills.db` 后删除：

- 仅在数据库损坏时使用
- 重置后下次同步从全量开始
- 备份文件保留 30 天

## 安全

### 修改启动密码

1. 进入 **设置 → 安全 → 修改启动密码**
2. 输入旧密码
3. 输入新密码（两次确认）
4. 应用重新加密 `identities.enc`

> ⚠️ 改密过程中**不要关闭应用**，否则数据可能损坏。

### 加密状态

显示当前 `identities.enc` 的：

- **算法**：AES-GCM-256
- **KDF**：PBKDF2-SHA256（100,000 轮）
- **文件大小**
- **最后修改时间**

### 重置加密（危险）

删除 `identities.enc`：

- 所有账号不可解密（必须重新添加）
- 账单数据 `bills.db` 保留
- 用于忘记启动密码时

## 日志

| 项 | 选项 |
|---|---|
| **日志级别** | Verbose / Debug / Information / Warning / Error |
| **文件轮转** | 按天 / 按大小 |
| **保留天数** | 默认 30 天 |
| **查看日志** | 打开 `logs/` 目录 |

### 日志文件

```
logs/
├── app-2026-06-03.log      当天日志
├── app-2026-06-02.log      历史日志
└── latest.log              最近一次启动的完整日志（不轮转）
```

## 关于

- **应用版本**：显示完整版本号
- **依赖版本**：列出关键 NuGet 包
- **更新日志**：点击查看 CHANGELOG
- **开源许可**：GPL-3.0
- **检查更新**：手动触发更新检查

## 配置文件位置

`config.toml` 文件位置（与 `Data/` 同级）：

- Windows: `%APPDATA%\shmtu-terminal-desktop\config.toml`
- Linux: `~/.config/shmtu-terminal-desktop/config.toml`
- macOS: `~/Library/Preferences/shmtu-terminal-desktop/config.toml`

直接编辑后保存，应用下次启动时自动加载。
