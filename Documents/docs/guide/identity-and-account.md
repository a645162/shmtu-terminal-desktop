# 身份与账号管理

应用支持**多身份**和**多账号**两级结构。本章说明字段含义、管理策略与典型场景。

## 数据模型

```
身份 (Identity)            账号 (Account)              CAS 凭证
┌─────────────────┐       ┌─────────────────────┐    ┌──────────────┐
│ ID              │ 1───* │ ID                  │    │ 学号          │
│ 显示名           │       │ 身份ID (FK)          │    │ 密码          │
│ 备注             │       │ 显示名               │    │ (加密存储)     │
│ 创建时间         │       │ 学号                 │    └──────────────┘
│                 │       │ 加密密码              │
└─────────────────┘       │ 同步模式              │
                          │ 验证码模式            │
                          │ 上次同步时间          │
                          │ 同步状态              │
                          └─────────────────────┘
```

## 身份 (Identity)

身份是对一个"使用场景"的封装，方便区分不同用途。

| 字段 | 必填 | 含义 | 例子 |
|---|---|---|---|
| **显示名** | 是 | 界面显示的名字 | "我"、"家人"、"同学 A" |
| **备注** | 否 | 自由描述 | "我自己" |
| **创建时间** | 自动 | 创建时记录 | - |

### 典型身份场景

- **个人身份**：装自己的校园账号
- **家庭身份**：装父母的账号方便查询消费（需对方授权）
- **同学身份**：临时帮人同步一次
- **测试身份**：开发/调试用

身份间数据**完全隔离**：账号、账单、配置都按身份分组。

## 账号 (Account)

账号隶属于一个身份，是 CAS 登录的凭证单位。

| 字段 | 必填 | 含义 |
|---|---|---|
| **显示名** | 是 | 账号在 UI 上的标签 |
| **学号** | 是 | CAS 用户名 |
| **密码** | 是 | CAS 密码（加密存储） |
| **同步模式** | 否 | 覆盖身份默认：`incremental` / `full` |
| **验证码模式** | 否 | 覆盖身份默认：`manual` / `remote` / `local` |
| **备注** | 否 | 自由描述 |

### 账号创建流程

1. 进入 **身份管理** → 选择/创建身份
2. 点击 **添加账号** → 填写显示名、学号、密码
3. 选择该账号的同步/验证码模式（或留空继承身份）
4. **保存** → 立即同步一次

## 凭证加密

账号的学号/密码使用 **AES-GCM-256** 加密，密钥由**启动密码 + 盐**通过 PBKDF2 派生。

```
明文 JSON:  { "accounts": [{ "id": "xxx", "username": "2024001", "password": "..." }] }
   ↓ AES-GCM 加密
identities.enc (二进制)
```

> ⚠️ **改密码或忘记启动密码**会导致所有账号无法解密，但账单数据库 `bills.db` 不受影响（明文）。请妥善保管启动密码。

## 增删改查

### 新建

UI：**身份管理** → **新建** / **添加账号**

### 编辑

修改显示名、备注、模式等。**学号/密码修改后需要重新登录 CAS**。

### 删除

- **删除身份**：级联删除其下所有账号和账单（**危险**）
- **删除账号**：保留账单记录，仅断开账号关联（用于"账号不再使用但账单想留"）

## 多账号并发同步

可在 **设置 → 同步 → 并发账号数** 配置同时同步的账号数（默认 1，建议 ≤ 3 避免触发 CAS 反爬）。

并发实现：`BillSyncService` 用 `SemaphoreSlim` 控制并发度，每账号独立进度推送：

```csharp
var throttler = new SemaphoreSlim(maxConcurrent);
await Task.WhenAll(accounts.Select(async acc =>
{
    await throttler.WaitAsync();
    try { await _sync.SyncAsync(acc, ...); }
    finally { throttler.Release(); }
}));
```

## 数据隔离与切换

身份切换时：

1. 重新加载 `identities.enc` 中对应身份的账号
2. 重新查询 `bills.db` 中该身份的账单
3. 触发 `MessageBus.IdentityChanged` 事件，所有 Tab VM 刷新

> 不同身份之间的账单**不会互相显示**。这避免家人/同学身份数据混淆。

## 最佳实践

- **测试身份**：日常用"个人身份"，开发用"测试身份"避免影响数据
- **定期清理**：删除不再使用的账号（保留孤儿账单）
- **导出备份**：身份修改后建议做一次快照
- **密码变更**：校园账号改密后必须更新应用内密码，否则下次同步会失败
